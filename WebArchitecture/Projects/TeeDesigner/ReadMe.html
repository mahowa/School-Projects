Summary:

 

Our project is product design application. The app gives the user the ability to create or upload a logo and then place it on a number of products or upload their own image and crop it so they can place their logo on it.   The people involved on this project were, Matt Howa, Matt Willden, and Charlie Barber.  We decided to use some open source libraries as our starting point and then add a number of features that we thought would improve the site.   We spent a large amount of time on this project.   Some of the time was spent individually working on our own parts but, we spent the majority of the time working together as a group.  We all ran into some difficulties in each of our implementations, but eventually overcame the obstacles and successfully deployed our individual parts to work together.

Summary

            I started by combining the two projects TShirtDesigner and method draw. I wanted to make method draw more like Adobe Illustrator so I built the layers panel. The layer’s panel adds a ton of functionality to the app giving the user the ability work on specific layers and also toggle their visibility, which in my experience is a huge feature in illustrator. I spent almost 15 hours between the two tasks.

            Next I want to make the TShirtDesigner more useful so I found a site that supplied a csv of product specifications and also images to go with the data. I imported the csv into my database then created API’s in order to extract the data needed to make the app work. From there I added a product dropdown that allows the user to select a product and load the image to the canvas. Many of the products came in different colors so I created a div that contained the thumbnails of each color. The thumbnails are clickable and once selected would load the specific color into the canvas.

            Finally, I spent the remaining hours assisting my group with their tasks and combining them in order to complete our project.

Matt Willden

 

  • Uploading Scalable Canvas Image

The purpose of this part was to create a way for the user to upload custom images to the canvas. With the image uploaded the user could then use the custom image as a template to see what other items look like. It was divided into 2 main parts discussed below.

 

            Part 1: Image uploader ~ 14 hrs.

 the first revolved around the basic image uploader. We decided to go with another open-source library called DropZonejs, it contained many additional features, such as custom event handlers and styling integration, validation etc... We could have written our own uploader but, wouldn’t have had all of the additional bonus features that this comes with it for free. The implementation was fairly easy with one caveat, we ran into a problem trying to get the permissions on the Azure Server to let us actually upload the images. This took a very long time to figure out. After that, it was only an hour or so to get the styling right. In the end it was worth the investment in dropzonejs. It gave the project a ton of functionality, without a ton of overhead once implanted. If other areas wanted to have the same uploader it would be easy for future integration and improvement.

 

Part 2: Image Cropper ~ 8 hrs

The second part focused taking what the image uploader uploaded, then making it fit the dimensions of the canvas i.e. “scalable”.  The idea was to be able to take the original image size, and help make it fit to the canvas size without distortion. Again, we chose to use an open-source javascript library to handle cropping. It came with a lot of features that we could leverage, like modifying a given element, then injecting it into a given canvas. It proved to be incredibly powerful, thought documentation was limited, so implementation was slow.

  • Keyboard Shortcuts

 

I spent around 4 hours on this particular feature.   I found a really cool javascript library extension called Mousetrap.  It was very easy and straightforward to use and helped me bind buttons to certain actions on the webpage.  Some of the features were very easy to implement.  I struggled with some of the features though.   I struggled to simulate clicking on the drop-down menus with a shortcut key.   I did not successfully implement this feature, but implemented a lot of the other buttons.  This was implemented on the main page of designing the shirt. 

I continued to implement more shortcuts on the graphic designer webpage.  Although it turned out to be simple,  it was very time consuming.  I ran into issues with the Mousetrap library I found and it would not work on this page despite working on the main page.  So then I researched and implemented the hot keys with the js.hotkeys.js library.   Once I figured out the syntax, I made it work very easily with the page.

 

  • Converting Image from Bitmap to SVG

I spent a considerable amount of time on this part of the project.   It totaled up to about 8 hours. The ReadMe for the library, Imagetracer.js, said that it could be implemented in Node.js.  So I spent a couple hours studying Node.js and was going to implement it with Node.js.   I discovered that you did not need to use Node.js and the way the library Imagetracer.js used it was very confusing.   I studied this library for quite awhile trying to figure out how it worked.  It was difficult to understand.   I ended up figuring out the basics of it and implemented that into our project so we could convert our images.   There were some more advanced tools in the library, but I could not figure them out.
